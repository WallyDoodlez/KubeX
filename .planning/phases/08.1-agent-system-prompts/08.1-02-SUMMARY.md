## Decisions Made

- Synchronous `httpx.get` used in `_fetch_worker_descriptions` — `_load_system_prompt` is called at startup before the asyncio event loop is in full operation for MCP; sync call avoids nested async complexity and is safe in this one-time startup path
- Self-filtering in worker list: orchestrator filters its own `agent_id` from the Registry response so it cannot accidentally delegate tasks to itself via the worker list section
- `test_config_no_skills_dir_uses_base_prompt` expectation updated (not deleted) — the test still validates the behavior, just against the new preamble-based output rather than the old constant string
