## Self-awareness Depth

### User-initiated decision: Capability-bounded behavior

User proposed (not from menu): All agents must reject requests outside their declared capabilities. No generic LLM behavior. Exception: liveness/identity queries ("who are you?", "are you alive?") — all agents respond with identity + capabilities.

**User's choice:** Capability-bounded agents with liveness exception
**Notes:** "We should add instruction for the kubexs to not become generic, and must only tend to requests bounded by the capabilities - but perhaps except for a liveness check." Also clarified orchestrator responds to whoami the same way.
