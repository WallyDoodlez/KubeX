    All endpoints require `Authorization: Bearer <KUBEX_MGMT_TOKEN>`. Document the header format. Note: default token for dev is `kubex-mgmt-token`.

    **4. API Contracts (per D-10):**
    For each endpoint, document:
    - HTTP method + path
    - Request body schema (JSON, with field types and descriptions)
    - Response schema (JSON, with status codes)
    - Example curl command
    - Error responses (status codes, error body format)

    Specific contracts:

    a) `GET /agents/{agent_id}/lifecycle` (SSE):
    - Response: `text/event-stream`
    - Event format: `data: {"agent_id": "string", "state": "string", "timestamp": "string"}\n\n`
    - Possible states: `booting`, `credential_wait`, `ready`, `busy`
    - No terminal event — stream runs until client disconnects
    - Error when Redis unavailable: `data: {"error": "Redis not available"}\n\n`
    - Curl: `curl -N -H "Authorization: Bearer kbx-token" http://gateway:8080/agents/my-agent/lifecycle`
    - **IMPORTANT NOTE:** Native browser `EventSource` API does NOT support custom headers. FE must use `fetch()` with `ReadableStream` or a library like `eventsource-parser` that supports custom headers. Include a JavaScript code example using `fetch` + `getReader()`.

    b) `POST /kubexes/{kubex_id}/credentials`:
    - Request: `{"runtime": "claude-code"|"gemini-cli"|"codex-cli", "credential_data": {...token JSON...}}`
    - Response 200: `{"status": "injected", "kubex_id": "...", "runtime": "...", "path": "..."}`
    - Response 404: `{"error": "KubexNotFound", "message": "..."}`
    - Response 422: `{"error": "UnknownRuntime", "message": "..."}`
    - Response 500: `{"error": "InjectionFailed", "message": "..."}`
    - Curl: `curl -X POST -H "Authorization: Bearer kbx-token" -H "Content-Type: application/json" -d '{"runtime":"claude-code","credential_data":{"token":"..."}}' http://manager:8081/kubexes/my-kubex/credentials`

    c) `GET /kubexes` and `GET /kubexes/{kubex_id}`:
    - Document the response schema including the `status` field
    - The `status` field reflects agent lifecycle state

    **5. Mermaid Sequence Diagram (per D-10):**
    Full OAuth provisioning flow:
    ```
    User -> Command Center UI: Click "Authorize Agent"
    UI -> Manager GET /kubexes: List agents, find target
    UI -> Gateway GET /agents/{id}/lifecycle: Open SSE stream
    UI: Show "Waiting for credentials" (state=credential_wait)
    User -> OAuth Provider: Complete OAuth flow in popup
    OAuth Provider -> UI: Return token
    UI -> Manager POST /kubexes/{id}/credentials: Inject token
    UI: SSE receives state=ready
    UI: Show "Agent Ready"
    ```

    **6. Edge Cases (per D-10):**
    - Token expiry mid-session: Agent enters CREDENTIAL_WAIT on next task attempt. SSE stream shows state transition. FE should show re-auth button. (This is AUTH-03 behavior — per D-09, the agent catches expired tokens at task execution time, not dispatch time. One failed task attempt may occur before the state change is visible.)
    - Container restart: Agent goes through booting -> credential_wait or booting -> ready depending on whether the named volume has valid credentials
    - Multiple agents: Each agent has its own lifecycle SSE stream. FE opens one EventSource per agent being monitored.
    - SSE reconnection: If the SSE connection drops, FE should reconnect. The stream will pick up from current state (no history replay).

    **7. Error Codes:**
    Table of all error codes across endpoints with recommended FE handling.

    All diagrams MUST use Mermaid syntax (per CLAUDE.md documentation standards). No ASCII art.
